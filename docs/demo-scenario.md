# 접수 면담 봇 — 데모 시나리오

이 문서는 의사 고객에게 intake-slot-engine 스키마 기반 문진 엔진을 시연하기 위한 대본이다.

## 준비: Fake 모드 구동

API 키 없이 데모를 실행하려면 fake 모드를 활성화한다.

```bash
# .env 파일 확인 (ANTHROPIC_API_KEY는 비워두거나 주석 처리)
# 또는 환경변수로 직접 지정

# Fake 모드로 구동
MODEL=fake KNOWLEDGE_DIR=knowledge .venv/bin/python -m uvicorn app.main:app --reload
```

Fake 모드 특징:
- API 호출 없음 (API 키 불필요)
- 모든 응답이 "[fake] 참고 문서: ..." 형태로 돌아옴
- 슬롯 추출만 정상 작동 (샘플 응답도 slot 추출 파이프라인을 거침)
- 10턴 한도와 최종 요약 JSON 출력은 그대로 작동

## 시연 순서

### 1단계: 시작 (Turn 1)

**시연자 발화**  
"안녕하세요. 오늘 뭐 때문에 오셨어요?"

**봇 응답 예상**  
```
[fake] 참고 문서: 초기-면담-목적과-구조, 비밀보장-원칙과-예외
```

**주목 포인트**  
- 개방형 질문으로 시작 (`_persona.md` 에 정의된 opening_question)
- 검색된 관련 문서: 초기 면담 구조와 비밀보장 설명

---

### 2단계: 정서 트랙 (Turns 2-3)

**시연자 발화**  
"최근에 자꾸 기분이 안 좋고 불안해요. 밤에 잠도 잘 못 자구요."

**봇 응답 예상**  
```
[fake] 참고 문서: 접수-면접-질문지-구성, 상담-목표-설정
| 채움: 상담 트랙=정서, 호소 문제=기분 저하·불안, 잠 문제
| 다음 질문: 증상 시기·일상 영향
```

**주목 포인트**  
- 슬롯이 채워짐: `track=정서`, `chief_complaint` 감지
- 불안/잠 신호가 정서 트랙으로 분류됨
- 우선순위에 따라 다음 질문 제시 (증상이 언제부터 시작되었는지)

**시연자 발화 (Turn 3)**  
"한 3개월 정도 되는 것 같아요. 일할 때도 집중이 안 돼요."

**봇 응답 예상**  
```
[fake] 참고 문서: 라포-형성-기법
| 채움: 증상 시기·일상 영향=3개월, 일상 기능 저하
| 다음 질문: 대처 시도
```

---

### 3단계: 관계 트랙 (Turns 4-5)

**시연자 발화 (Turn 4)**  
"남편하고도 요즘 자주 싸워요. 남편은 내가 너무 예민하다고 해요."

**봇 응답 예상**  
```
[fake] 참고 문서: 비밀보장-원칙과-예외, 라포-형성-기법
| 채움: 지지체계=남편
| 다음 질문: 관계 대상·기간
```

**주목 포인트**  
- 새 트랙 신호 감지 (관계 관련 "남편", "싸워요")
- 기존 정서 트랙과 함께 관계 정보도 누적
- 슬롯 계층화: 공통 슬롯(`track`, `chief_complaint`) + 트랙별 조건부 슬롯

**시연자 발화 (Turn 5)**  
"우린 12년 됐는데 최근 1년 사이 멀어진 것 같아요."

**봇 응답 예상**  
```
[fake] 참고 문서: 상담-목표-설정
| 채움: 관계 대상·기간=남편 12년, 최근 1년 악화
| 다음 질문: 상담 기대
```

---

### 4단계: 위기 트랙 + 레드플래그 우선 질문 (Turns 6-7)

**시연자 발화 (Turn 6)**  
"요즘 너무 힘들어서 아예 죽고 싶다는 생각까지 들어요."

**봇 응답 예상**  
```
[fake] 참고 문서: 위기-상황-스크리닝, 비밀보장-원칙과-예외
| 채움: 상담 트랙=위기
| 다음 질문: 자해·자살 시도 이력
```

**주목 포인트**  
- "죽고 싶" 신호가 위기 트랙을 활성화
- **레드플래그 우선화**: 다음 슬롯이 `crisis_plan_means` (계획·수단)가 아니라 먼저 시도 이력을 묻는다 → 위험도 평가 후 개입 결정
- `red_flag: true` 슬롯은 우선순위 6으로 높아지지만, 먼저 맥락 파악

**시연자 발화 (Turn 7)**  
"한번 약을 많이 먹으려고 한 적이 있는데 용기가 안 났어요."

**봇 응답 예상**  
```
[fake] 참고 문서: 위기-상황-스크리닝
| 채움: 자해·자살 시도 이력=시도 계획 있음 (미완료)
| 다음 질문: 자해 계획·수단 [★ RED FLAG]
```

**주목 포인트**  
- 레드플래그 슬롯이 이제 다음 질문으로 떠오름
- 비밀보장 예외 상황(즉각적 위해) 문서 자동 참고

---

### 5단계: 혼합 발화 (다중 슬롯 동시 채움) (Turn 8)

**시연자 발화 (Turn 8)**  
"실은 이 1주일 동안 계속 이런 생각이 들어요. 친구한테 좀 얘기했는데 혼자라고 느껴져요."

**봇 응답 예상**  
```
[fake] 참고 문서: 라포-형성-기법, 상담-목표-설정
| 채움: 자해 계획·수단=구체적 계획 있음, 지지체계=친구 (미충분)
| 다음 질문: 상담 기대
```

**주목 포인트**  
- 한 발화에서 여러 슬롯 동시 채움 (일반적인 상담 흐름)
- 계획·수단 세부사항을 인지했지만 fake 모드라 분석 미세한 추출 없음
- 지지체계 업데이트 (친구 추가)

---

### 6단계: 마무리 (Turn 9-10)

**시연자 발화 (Turn 9)**  
"제 상담을 받아볼 수 있을까요? 지금 정신이 좀 오락가락해요."

**봇 응답 예상**  
```
[fake] 참고 문서: 상담-목표-설정
| 채움: 상담 기대=상담 신청 의지 있음, 정신 상태 불안정

```

**시연자 발화 (Turn 10 — 최종)**  
"네, 가능하면 이번 주에 시간을 내서 와 보겠습니다."

**봇 응답**  
```
[fake] 참고 문서: 상담-목표-설정
(Turn 10/10 — 한도 도달)
```

## 재활병원 프리뷰 포인트 (CAP15)

현재 시연 중인 3-트랙 구조(정서/관계/위기)는 실제 재활병원 버전의 프리뷰다:

| 상담 (현재) | 재활 병원 (계획) |
|----------|--------------|
| 정서 트랙 | 암재활 (정서/심리 재활) |
| 관계 트랙 | 근골격 (가족 지지 맥락) |
| 위기 트랙 | 자율신경 (즉시 의료 개입) |

**시연 중 언급할 지점:**  
"이 3가지 분류 방식은 나중에 암 환자 재활, 뇌졸중 환자 근골격 재활, 자율신경실조 환자 응급 대응으로 그대로 확장되는 프레임워크입니다."

## 지식 교체 시연 (CAP19)

같은 코드가 완전히 다른 도메인에서도 동작함을 보여준다.

### 시연 환경 전환

첫 번째 세션 완료 후, 새 터미널에서:

```bash
# 지식셋을 커피 Q&A로 교체
MODEL=fake KNOWLEDGE_DIR=knowledge-alt .venv/bin/python -m uvicorn app.main:app --reload --port 8001
```

### 커피 도메인 시연 (짧은 버전)

**시연자 발화**  
"커피를 잘 못 내려요. 어떻게 배워야 해요?"

**봇 응답 예상**  
```
[fake] 참고 문서: 드립-커피-추출-원리, 원두-보관법
```

**주목 포인트**  
- 슬롯 스키마(`_intake_schema.md`)가 없어서 기본 RAG 모드로 동작
- 같은 검색·참고 파이프라인이 상담(structured) → 커피 Q&A(unstructured) 로 매끄럽게 전환
- API 키 없이도 즉시 실행 가능 → 출시 전 데모 및 고객 proof-of-concept 용

## 민감정보 주의

위기 트랙 응답(자살 시도 이력, 계획, 수단)은 `data/conversations/{date}/{session_id}.json` 에 평문 JSON으로 저장된다.

**데모 시연 시 주의:**
- 실제 자살/자해 고위험 사례는 절대 사용 금지
- 모든 내용은 가상의 임상 사례로만 시연할 것
- 데모 세션 데이터는 시연 직후 `data/conversations/` 에서 삭제
- 실 운영 환경에서는 민감정보 암호화(Phase 7 보안 검토 후속 항목) 필요

## 체크리스트

시연자가 확인해야 할 항목:

- [ ] Fake 모드 구동 성공 (MODEL=fake, API 키 불필요)
- [ ] Turn 1: 개방형 질문 시작 확인
- [ ] Turn 2-3: 정서 트랙 슬롯 채움 확인
- [ ] Turn 4-5: 관계 트랙 조건부 슬롯 활성화 확인
- [ ] Turn 6-7: 위기 신호 감지 + 레드플래그 우선화 확인
- [ ] Turn 8: 혼합 발화에서 다중 슬롯 동시 채움 확인
- [ ] Turn 9-10: 한도 도달 및 요약 JSON 생성 확인
- [ ] 재활 트랙 맵핑 설명 (정서/관계/위기 → 암재활/근골격/자율신경)
- [ ] 지식 교체 시연 (knowledge-alt로 커피 봇으로 전환)
- [ ] 민감정보 주의사항 안내 완료
