# 수학 학습 코칭 접수 슬롯 스키마

PNK 수학 방법론 기반 학습 코칭에 앞서, 학생의 현재 상태를 파악하는 접수
슬롯을 선언한다. 트랙(위기/개념/문제풀이/학습습관) 판별이 최우선 슬롯이고,
공통 슬롯과 트랙별 조건부 슬롯이 뒤따른다. 위기 트랙과 그 안전 확인 슬롯은
상담 도메인 스키마(`knowledge/_intake_schema.md`)에서 그대로 가져왔다 —
학생 대상 봇에서도 자해·자살 신호 감지와 안전 확인은 도메인과 무관하게
유지한다.

근거: `PNK수학의 방향성.md`(의식적 논리 사고 4과정),
`문제풀이4단계(고난도 문제를 풀기 위한 필수단계).md`(읽기→기억→이해→풀이),
`Why&How(제대로 된 셀프 피드백).md`(오답 피드백 습관).

## 트랙 매핑과 우선순위

**판정 우선순위: 위기 > 개념 > 문제풀이 > 학습습관** (`signals` 선언 순서 =
매칭 순서, 엔진 `_match_signal`이 첫 매칭에서 확정)

| 우선순위 | 트랙 | 대표 호소 | 처리 원칙 |
|---|---|---|---|
| 1 | 위기 | 자해, 자살 사고 | 다른 신호와 겹치면 안전 확인을 최우선 |
| 2 | 개념 | 설명을 들어도 이해가 안 됨, 정의·공식·원리가 낯섦 | 어디까지 이해되는지 경계 확인 |
| 3 | 문제풀이 | 문제에 손도 못 댐, 접근법을 모름, 고난도에서 막힘 | 문제풀이 4단계 중 막히는 지점 확인 |
| 4 | 학습습관 | 실수 반복, 시험 불안, 시간 부족, 공부 방법 고민 | 상황·습관 맥락 확인 |

## 판단 기록 (관리자 확인용)

모호한 지점에서 AI가 판단하고 진행한 내역. 도메인 관점에서 재검토·조정 대상.

- **2026-07-16 / 위기 트랙·안전 슬롯 유지**: 수학 코칭 봇이지만 대상이
  학생이라 자해·자살 신호 감지와 안전 확인(자살예방상담전화 109 등)을 상담
  스키마에서 그대로 가져왔다. 신호어 사전·reject 가드·override 규칙 모두 동일.
  앱 레벨 중독 라우팅(`app/addiction.py` — 게임·인터넷 과의존 포함)도 스키마와
  무관하게 동작하므로 학생이 게임 중독 호소를 하면 전문기관 안내로 연결된다.
- **2026-07-16 / 트랙 4종 설계**: PNK 방법론의 진입점에 맞춰 개념(이해 경계
  확인) / 문제풀이(문제풀이 4단계 중 막히는 지점) / 학습습관(Why&How 피드백
  습관)으로 나눴다. stuck_point의 값 4종은 `문제풀이4단계` 문서의
  읽기→기억→이해→풀이 단계를 학생 언어(문제 이해/접근 방법/계산/시간)로
  번역한 것이다.
- **2026-07-16 / math_topic 신호어에서 "계산" 제외**: "계산 실수를 자꾸 해요"
  (학습습관 호소)가 수와 연산 단원으로 오분류되는 부분문자열 충돌이 있어
  수와 연산 신호를 분수·사칙연산 등 명시적 단원 어휘로 좁혔다.
- **2026-07-16 / expectation 슬롯 id 유지**: 스테퍼 단계 파생 규칙
  (`static/app.js`의 `lmwikiDeriveStep`)이 마지막 단계 판정에 `expectation`
  id를 쓰므로 도메인이 바뀌어도 마지막 슬롯 id는 `expectation`으로 유지한다.
- **2026-07-16 / ui 섹션 신설**: 화면 고정 문구(제목·인사말·프라이버시
